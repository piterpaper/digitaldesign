01

Typescript

Это язык программирования, представленный Microsoft в 2012 году и позиционируемый как средство разработки веб-приложений. По сути это типизированное надмножество JavaScript, предназначенное для выявления ошибок на этапе компиляции. Программа на TypeScript компилируется в простой код JavaScript, который выполняется в любом браузере.

Плюсы TypeScript:

  • Ошибки, допущенные в процессе модификации кода, видны сразу, а не во время выполнения
  • В TypeScript есть интерфейсы и классы
  • Имеет открытый исходный код, доступный на GitHub
  • TypeScript имеет высокую скорость компиляции и распространяется по лицензии Apache
  • TypeScript позволяет быстрее и проще писать комплексные решения, которые в дальнейшем будет легче развивать и тестировать

Минусы TypeScript

  • Невозможно типизировать ошибки, появившиеся в режиме реального времени
  • Не все браузеры поддерживают вкладку TypeScript в консоли, что приводит к большим временным затратам
  • Несмотря на статическую типизацию, можно допустить прохождение любого типа данных, прописав слово any

02

PWA

Progressive Web Applications — впервые предложенный в 2018 году подход к разработке быстрых, надежных и удобных для пользователя веб-приложений. Поисковая система Google отделяет веб-приложения от нативных именно по этим критериям, поэтому в 2022 году подход будет набирать популярность.

Чтобы приложение можно было назвать прогрессивным, в нем должен использоваться определенный набор технологий:

  • Service Worker — прокси-слой между фронтендом и бэкэндом, через который проходят все запросы браузера
  • HTTPS — подход предполагает, что все ресурсы сайта передаются через защищенный протокол. Кроме того, на сайте не должно быть ссылок на незащищенные ресурсы;
  • App shell — основа графического интерфейса, которая хранится на стороне клиента и загружается при запуске веб-приложения;
  • Web App manifest — определяет, как будет выглядеть название приложения, его иконка и само PWA в браузере;
  • Push-уведомления — те самые всплывающие окошки с предложением получать новости от сайта. Единственная широко используемая технология PWA, которой многие владельцы сайтов злоупотребляют

03

Бессерверная архитектура

Бессерверная архитектура - еще одна развивающаяся в 2021 году веб-технология. Большинство организаций выбрало бессерверную архитектуру в качестве основного рабочего решения. В 2021 году количество таких решений будет расти из-за ситуации с пандемией. Из-за распространения Covid -19 многие организации (большие, средние и маленькие) могут использовать свои ресурсы из дома. Эта новая тенденция позволила быстро использовать бессерверную архитектуру или облачные решения.

Согласно недавнему исследованию, в настоящее время 98 процентов организаций используют облачные сервисы. Это доказывает, что облачная архитектура - одна из главных особенностей веб-разработки. Бессерверная архитектура предлагает несколько новых функций в процессе развития веб-технологий. Теперь люди используют бессерверные вычисления, бессерверные бухгалтерские решения, быстрее и обновляют коэффициент, обеспечивают экономически эффективные решения, обслуживая делегированные проблемы.

Благодаря бессерверной архитектуре организации могут понимать SaaS, Laas, PaaS и т.д. Бессерверная архитектура предлагает веб-приложения без необходимости думать о серверах. Облачная структура может открываться в любом месте, где есть доступ в Интернет. Даже можно хранить тонны информации без использования выделенных серверов. В нынешней ситуации большинство организаций без колебаний используют эту облачную систему. Он полностью изменил веб-технологии и концепцию реального управления серверами.
